# Agentic Sessions Manager

## Goals

- provide a simple interface for managing agentic sessions
- probably something running in the background that can be queried for status and results

## what should workflow look like

- multiple (tmux / vscode / ...) sessions running in the background, each with a different agentic session
- they can run claude or opencode or something else
- when agent is waiting for input or is finished with a task, there should be

, it should be possible to query the session manager for the status of the session and the results so far
